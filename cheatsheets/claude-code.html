일반 제어
Ctrl+C
현재 진행 중인 입력이나 Claude의 응답 생성을 즉시 중단합니다. 긴 작업을 멈출 때 유용합니다.
입력 / 생성 취소
Ctrl+D
현재 Claude Code 세션을 완전히 종료합니다. 대화 내용은 자동 저장되므로 나중에 재개 가능합니다.
세션 종료
Ctrl+L
터미널 화면을 깨끗이 지웁니다. 대화 컨텍스트는 유지됩니다.
화면 지우기
Ctrl+O
도구 호출의 세부 출력을 표시하거나 숨깁니다. 디버깅 시 어떤 도구가 어떻게 호출되는지 확인할 때 유용합니다.
상세 출력 전환
Ctrl+R
이전 프롬프트 입력 이력을 역순으로 검색합니다. bash의 reverse-search와 동일한 방식입니다.
히스토리 검색
Ctrl+G
$EDITOR에 설정된 외부 편집기(VS Code, vim 등)에서 긴 프롬프트를 작성할 수 있습니다. 복잡한 지시사항 작성 시 편리합니다.
프롬프트 편집기 열기
Ctrl+B
현재 작업을 백그라운드로 보냅니다. 터미널을 닫아도 작업이 계속되며, 나중에 Ctrl+T로 확인하거나 재개할 수 있습니다.
백그라운드 실행
Ctrl+T
백그라운드에서 실행 중인 작업 목록을 표시하거나 숨깁니다. 병렬 작업 관리에 유용합니다.
작업 목록 전환
Ctrl+V
클립보드의 이미지를 대화에 붙여넣습니다. Mac에서도 Cmd+V가 아닌 Ctrl+V입니다. 스크린샷 리뷰나 UI 분석에 활용됩니다.
이미지 붙여넣기
Ctrl+X
Ctrl+K
실행 중인 백그라운드 에이전트를 강제 종료합니다. 두 키 조합을 순서대로 입력합니다.
백그라운드 에이전트 종료 NEW
Esc×2
Esc를 두 번 눌러 마지막 작업을 되돌립니다. 체크포인트로 롤백하여 다른 접근을 시도할 수 있습니다.
되돌리기 (Undo)
모드 전환
Shift+Tab
Normal → Auto → Plan 모드를 순환합니다. Plan 모드에서는 코드 변경 없이 계획만 수립하고, Auto 모드에서는 권한 확인 없이 자동 실행합니다.
권한 모드 순환
Alt+P
Haiku, Sonnet, Opus 등 사용 가능한 모델 사이를 전환합니다. 작업 복잡도에 따라 적절한 모델을 선택하세요.
모델 전환
Alt+T
Extended Thinking을 켜거나 끕니다. 활성화하면 Claude가 응답 전에 내부적으로 더 깊이 사고합니다. 복잡한 문제에 유용합니다.
사고 모드 전환
입력 제어
⏎ Enter
백슬래시(\) + Enter로 프롬프트 내에서 줄을 바꿉니다. 여러 줄의 지시사항을 입력할 때 사용합니다.
빠른 줄바꿈
Ctrl+J
줄바꿈을 삽입하는 대체 방법입니다. 일부 터미널에서 \Enter가 동작하지 않을 때 사용하세요.
제어 시퀀스 줄바꿈
접두사
/
내장 명령어를 실행합니다. /clear, /compact, /model 등 다양한 명령어를 사용할 수 있으며, 자동완성을 지원합니다.
슬래시 명령어
!
터미널 명령어를 Claude를 거치지 않고 직접 실행합니다. 예: !git status, !npm test. 결과는 대화 컨텍스트에 포함됩니다.
직접 bash 실행
@
파일을 직접 참조합니다. @를 입력하면 파일명 자동완성이 시작됩니다. 컨텍스트에 특정 파일을 명시적으로 포함시킬 때 유용합니다.
파일 언급 + 자동완성

↑↓
세션 목록에서 위/아래로 이동합니다.
항목 탐색
←→
세션 그룹을 확장하거나 축소합니다.
확장 / 축소
P
선택한 세션의 내용을 미리 볼 수 있습니다.
미리보기
R
선택한 세션의 이름을 변경합니다.
이름 변경
/
세션 목록에서 키워드로 검색합니다.
검색
A
모든 프로젝트의 세션을 표시합니다.
전체 프로젝트
B
현재 git 브랜치의 세션만 필터링하여 표시합니다.
현재 브랜치
세션 관련
/clear
현재 대화의 모든 컨텍스트를 삭제합니다. 새로운 대화를 시작하고 싶을 때 사용하세요.
대화 내용 초기화
/compact
대화 내용을 요약하여 토큰을 절약합니다. /compact [focus]로 특정 주제에 집중하여 압축할 수도 있습니다. 컨텍스트가 95% 찰 때 자동 실행됩니다.
컨텍스트 압축
/resume
이전 세션 목록을 표시하고, 선택한 세션으로 전환합니다. 세션 선택기 단축키와 함께 사용합니다.
세션 재개 / 전환
/rename
현재 세션에 알아보기 쉬운 이름을 붙입니다. 나중에 claude -r "name"으로 재개할 때 편리합니다.
세션 이름 변경
/branch
현재 대화에서 새 브랜치를 생성합니다. 다른 접근 방식을 시도할 때 원래 대화를 보존하면서 분기할 수 있습니다.
대화 분기
/cost
현재 세션의 토큰 사용량과 비용을 표시합니다.
토큰 사용량 통계
/context
현재 사용 중인 컨텍스트를 그리드 형태로 시각화합니다. 어떤 파일과 대화가 얼마나 공간을 차지하는지 한눈에 확인할 수 있습니다.
컨텍스트 시각화
/diff
현재 세션에서 변경된 파일들의 diff를 인터랙티브하게 보여줍니다.
인터랙티브 diff 뷰어
/copy
마지막 응답을 클립보드에 복사합니다. /copy N으로 N번째 응답을 지정할 수도 있습니다.
응답 복사
/rewind
대화를 이전 체크포인트로 되돌립니다. 잘못된 방향으로 진행됐을 때 유용합니다.
되돌리기 / 체크포인트
/export
현재 대화를 파일로 내보냅니다. 문서화나 공유에 활용할 수 있습니다.
대화 내보내기
설정 관련

Claude Code 설정 파일을 열어 편집할 수 있습니다.
설정 열기
/model
사용할 모델을 변경합니다. /model [model-name]으로 직접 지정하거나, 목록에서 선택합니다.
모델 전환
/fast
같은 모델에서 더 빠른 출력 속도를 제공합니다. 모델이 변경되는 것이 아니라 동일 모델의 빠른 버전을 사용합니다.
빠른 모드 전환
/vim
입력 라인에서 Vim 키바인딩을 활성화/비활성화합니다.
Vim 모드 전환
/theme
Claude Code의 터미널 색상 테마를 변경합니다.
색상 테마 변경
/permissions
현재 도구 권한 설정을 확인하고 수정합니다. 어떤 도구가 자동 승인되어 있는지 관리할 수 있습니다.
권한 조회/수정
/effort
Claude의 응답 깊이를 조절합니다. low(빠른 답변), med(기본), high(심층 분석), max(최대 사고), auto(자동) 중 선택합니다.
노력 수준 설정 NEW
/color
하단 프롬프트 바의 색상을 변경합니다. 여러 세션을 구분할 때 유용합니다.
프롬프트 바 색상
/keybindings
키보드 단축키를 사용자 지정으로 변경할 수 있습니다. ~/.claude/keybindings.json 파일을 편집합니다.
키보드 단축키 커스터마이즈
/terminal-setup
터미널 에뮬레이터의 키바인딩이 Claude Code와 호환되도록 설정합니다.
터미널 키바인딩 설정
도구 관련
/init
프로젝트에 CLAUDE.md 파일을 생성합니다. 프로젝트 규칙, 코딩 스타일, 아키텍처 설명 등을 담을 수 있습니다.
CLAUDE.md 생성
/memory
CLAUDE.md 파일을 편집기에서 열어 프로젝트 지시사항을 수정합니다.
CLAUDE.md 파일 편집

Chrome 연동
/reload-plugins

플러그인 핫 리로드
/add-dir
추가 작업 디렉토리를 현재 세션에 포함합니다. 여러 프로젝트를 동시에 참조할 때 유용합니다.
워킹 디렉토리 추가
특수 명령
/btw
현재 대화 컨텍스트를 소비하지 않고 별도의 짧은 질문을 할 수 있습니다. 간단한 문법 확인 등에 적합합니다.
별도 질문 (컨텍스트 비용 없음)
/plan
코드 변경 없이 계획만 수립하는 모드를 시작합니다. 복잡한 작업 전에 전략을 세울 때 사용하세요.
계획 모드 시작
/loop
/loop 5m "메시지"처럼 일정 간격으로 작업을 반복 실행합니다. 배포 모니터링이나 주기적 확인에 활용됩니다.
반복 작업 예약
/voice
Push-to-talk 음성 입력을 활성화합니다. 스페이스바를 누르고 말한 후 놓으면 전송됩니다. 한국어 포함 20개 언어를 지원합니다.
음성 입력 (20개 언어 지원)
/doctor
Claude Code 설치 상태를 진단하고 문제가 있으면 해결 방법을 제안합니다.
설치 진단
/stats
사용 빈도, 스트릭, 선호도 등의 통계를 보여줍니다.
사용 통계 & 패턴
/insights

로컬 프로세스

원격 SSE

프로젝트 (팀 공유/VCS)

홈 디렉토리의 ~/.claude.json에 저장됩니다. 모든 프로젝트에서 공통으로 사용할 MCP 서버를 여기에 설정합니다.
사용자 (전역)

Claude Code 자체를 MCP 서버로 실행하여 다른 클라이언트에서 접속할 수 있게 합니다.
CC를 MCP 서버로 실행
Elicitation Servers: 작업 중 사용자 입력을 요청하는 기능 NEW
CLAUDE.md 위치
./CLAUDE.md
프로젝트 루트에 위치합니다. 팀 전체의 코딩 규칙, 아키텍처 결정, 프로젝트별 지시사항을 담습니다. Git에 커밋하여 공유합니다.
프로젝트 (팀 공유)
~/.claude/CLAUDE.md
홈 디렉토리의 개인 설정입니다. 선호하는 코딩 스타일, 언어 설정 등 모든 프로젝트에 공통 적용할 지시사항을 담습니다.
개인 (모든 프로젝트)
/etc/claude-code/
시스템 관리자가 조직 전체에 적용하는 규칙입니다. 보안 정책이나 조직 공통 가이드라인을 담습니다.
조직 관리 (org-wide)
규칙 & 임포트
.claude/rules/*.md
프로젝트별 규칙 파일들입니다. 파일별로 규칙을 분리하여 관리할 수 있습니다. paths: 프론트매터로 특정 경로에만 적용되도록 설정 가능합니다.
프로젝트 규칙
~/.claude/rules/*.md
개인 전역 규칙 파일들입니다. 모든 프로젝트에 적용됩니다.
사용자 규칙
@path/to/file
CLAUDE.md 내에서 @구문으로 다른 파일의 내용을 임포트할 수 있습니다. 규칙을 모듈화하여 관리할 때 유용합니다.
CLAUDE.md에서 파일 임포트
paths: frontmatter
규칙 파일의 프론트매터에 paths를 지정하면 해당 경로의 파일을 작업할 때만 규칙이 적용됩니다.

자동 메모리
