$ Linux Commands Cheat Sheet

Linux 실무 필수 명령어
검색 결과가 없습니다.
기본 이동 & 조회
pwd
현재 위치한 디렉토리의 전체 경로를 출력합니다.
현재 작업 디렉토리 출력
cd <경로>
cd ~ (홈), cd .. (상위), cd - (이전 디렉토리)로 빠르게 이동할 수 있습니다.
디렉토리 이동
ls
현재 디렉토리의 파일과 폴더를 나열합니다. 기본적으로 숨김 파일은 표시하지 않습니다.
파일/디렉토리 목록 조회
ls -al
-a는 숨김 파일(.으로 시작), -l은 권한/소유자/크기/날짜 등 상세 정보를 표시합니다.
상세 목록 (숨김 파일 포함)
ls -lh
-h 옵션으로 파일 크기를 KB, MB, GB 단위로 표시합니다. 예: 4.0K, 1.2M
사람이 읽기 쉬운 크기 표시
파일 / 디렉토리 조작
mkdir <이름>
mkdir -p a/b/c 로 중간 디렉토리를 한 번에 생성할 수 있습니다.
디렉토리 생성
mkdir -p <경로>
부모 디렉토리가 없어도 자동으로 생성합니다. 스크립트에서 자주 사용합니다.
중간 경로 포함 생성
touch <파일>
파일이 없으면 새로 만들고, 있으면 수정 시간만 업데이트합니다.
빈 파일 생성 / 수정시간 갱신
cp <원본> <대상>
cp -r 옵션으로 디렉토리를 재귀적으로 복사합니다. 예: cp file.txt file_backup.txt
파일 복사
cp -r <원본> <대상>
디렉토리와 하위 내용을 모두 복사합니다. 백업 시 자주 사용합니다.
디렉토리 재귀 복사
mv <원본> <대상>
mv old.txt new.txt (이름변경), mv file.txt /tmp/ (이동). 같은 명령어로 두 가지 용도로 사용합니다.
파일/디렉토리 이동 또는 이름 변경
rm <파일>
휴지통 없이 바로 삭제됩니다! rm -i 옵션으로 삭제 전 확인을 받을 수 있습니다.
파일 삭제
rm -rf <경로>
-r(재귀) -f(강제) 삭제. 매우 위험한 명령어입니다. 실행 전 경로를 반드시 확인하세요! 특히 rm -rf / 는 절대 실행하지 마세요.
디렉토리 강제 삭제 주의
ln -s <원본> <링크>
바로가기와 비슷합니다. 원본 파일을 가리키는 링크를 만듭니다. 예: ln -s /usr/local/bin/node node
심볼릭 링크 생성
검색
find <경로> -name "<패턴>"
find . -name "*.log" (현재 디렉토리에서 .log 파일 찾기). -type f(파일만), -type d(디렉토리만) 옵션도 유용합니다.
파일 찾기
find <경로> -mtime -<일>
find . -mtime -7 (7일 이내 수정된 파일). -mtime +30 (30일 이전 파일) 등으로 오래된 파일 정리에 활용합니다.
최근 수정된 파일 찾기
which <명령어>
which node, which python 등으로 어떤 버전의 바이너리가 실행되는지 확인할 때 사용합니다.
실행 파일 경로 확인
파일 보기
cat <파일>
짧은 파일을 빠르게 확인할 때 사용합니다. 긴 파일은 less를 사용하세요.
파일 전체 내용 출력
less <파일>
방향키/Page Up·Down으로 이동, /검색어로 검색, q로 종료. 로그 파일 볼 때 필수입니다.
페이지 단위로 파일 보기
head -n <줄수> <파일>
head -n 20 app.log (처음 20줄). -n 생략 시 기본 10줄을 표시합니다.
파일 처음 N줄 출력
tail -n <줄수> <파일>
tail -n 50 app.log (마지막 50줄). 로그 파일의 최근 내용 확인에 자주 사용합니다.
파일 마지막 N줄 출력
tail -f <파일>
로그 파일을 실시간으로 모니터링합니다. Ctrl+C로 종료. 서버 로그 디버깅 시 필수입니다.
실시간 파일 추적
wc -l <파일>
wc -w (단어 수), wc -c (바이트 수). 파이프와 함께: cat file | wc -l
줄 수 카운트
텍스트 검색 & 처리
grep "<패턴>" <파일>
grep -i (대소문자 무시), grep -r (재귀 검색), grep -n (줄 번호 표시). 예: grep -rn "TODO" src/
텍스트 검색
grep -r "<패턴>" <경로>
프로젝트 전체에서 특정 문자열을 찾을 때 사용합니다. grep -rn "import express" . --include="*.js"
디렉토리 재귀 검색
sort <파일>
sort -n (숫자 정렬), sort -r (역순), sort -u (중복 제거 후 정렬). 파이프와 함께 많이 사용합니다.
정렬
uniq
보통 sort와 함께 사용: sort file.txt | uniq -c (중복 횟수 표시). 로그 분석에 유용합니다.
연속 중복 제거
diff <파일1> <파일2>
두 파일의 차이점을 줄 단위로 보여줍니다. diff -u (unified 형식)로 보면 더 읽기 쉽습니다.
두 파일 비교
awk '{print $N}'
공백으로 구분된 텍스트에서 N번째 컬럼을 추출합니다. 예: ps aux | awk '{print $2, $11}' (PID와 명령어)
컬럼 추출
sed 's/찾기/바꾸기/g'
sed -i 's/old/new/g' file.txt (파일 직접 수정). 스크립트에서 설정값 변경 시 유용합니다.
텍스트 치환
xargs
find . -name "*.tmp" | xargs rm (찾은 파일을 삭제). 다른 명령어의 출력을 인자로 전달할 때 사용합니다.
파이프 입력을 인자로 변환
권한 관리
chmod <권한> <파일>
chmod 755 script.sh (소유자:rwx, 그룹:r-x, 기타:r-x). chmod +x script.sh (실행 권한 추가). 숫자 표기: r=4, w=2, x=1
파일 권한 변경
chmod -R <권한> <경로>
chmod -R 644 logs/ (디렉토리 하위 모든 파일의 권한을 변경합니다)
디렉토리 하위 전체 권한 변경
chown <유저>:<그룹> <파일>
chown www-data:www-data /var/www (웹 서버 디렉토리 소유자 변경). -R 옵션으로 재귀 적용 가능합니다.
소유자 변경
umask
새로 생성되는 파일/디렉토리의 기본 권한을 결정합니다. umask 022가 일반적입니다.
기본 권한 마스크 확인/설정
권한 숫자 읽기: r(읽기)=4, w(쓰기)=2, x(실행)=1 → 합산. 예: 755 = rwxr-xr-x, 644 = rw-r--r--
파일 정보
ls -la
drwxr-xr-x 형태로 표시됩니다. d=디렉토리, r=읽기, w=쓰기, x=실행. 첫 번째 3자리=소유자, 두 번째=그룹, 세 번째=기타
권한 상세 확인
stat <파일>
파일 크기, 블록 수, inode, 접근/수정/변경 시간, 권한 등 모든 메타데이터를 표시합니다.
파일 상세 메타데이터
file <파일>
확장자에 의존하지 않고 파일의 실제 타입을 판별합니다. 예: file image.png → PNG image data
파일 타입 확인
프로세스 조회
ps aux
a(모든 사용자), u(상세 정보), x(터미널 없는 프로세스 포함). CPU, 메모리 사용량, PID 등을 확인할 수 있습니다.
전체 프로세스 목록
ps -ef
-e(모든 프로세스), -f(full format). PPID(부모 PID)를 확인할 수 있어 프로세스 관계를 파악하기 좋습니다.
전체 프로세스 (다른 형식)
ps aux | grep <이름>
ps aux | grep node (node 프로세스 찾기). grep -v grep을 추가하면 grep 자체를 제외할 수 있습니다.
특정 프로세스 검색
top
CPU/메모리 사용량 순으로 프로세스를 실시간 표시합니다. q로 종료. Shift+M(메모리 순), Shift+P(CPU 순) 정렬.
실시간 프로세스 모니터링
htop
top의 개선 버전으로 컬러 표시, 마우스 지원, 트리 뷰 등을 제공합니다. 설치 필요: apt install htop
향상된 프로세스 모니터링
프로세스 제어
kill <PID>
프로세스에 정상 종료 신호(SIGTERM)를 보냅니다. PID는 ps 명령어로 확인할 수 있습니다.
프로세스 종료 (SIGTERM)
kill -9 <PID>
정상 종료가 안 될 때 사용합니다. 프로세스가 정리 작업을 할 수 없으므로 최후의 수단으로 사용하세요.
프로세스 강제 종료 (SIGKILL) 주의
killall <이름>
killall node (모든 node 프로세스 종료). 여러 프로세스를 한 번에 종료할 때 편리합니다.
이름으로 프로세스 종료
nohup <명령어> &
nohup node server.js & → 터미널을 닫아도 서버가 계속 실행됩니다. 출력은 nohup.out에 저장됩니다.
백그라운드 실행 (터미널 종료 후 유지)
jobs
현재 셸에서 실행 중인 백그라운드 작업을 보여줍니다.
백그라운드 작업 목록
lsof -i :<포트>
lsof -i :3000 (3000번 포트 사용 프로세스). 포트 충돌 시 어떤 프로세스가 점유하고 있는지 확인할 때 필수입니다.
특정 포트를 사용 중인 프로세스 확인
네트워크 조회
ip addr ifconfig
네트워크 인터페이스별 IP 주소, MAC 주소 등을 확인합니다. 최근 리눅스에서는 ip addr 사용을 권장합니다.
IP 주소 확인
ping <호스트>
ping google.com (연결 상태 확인). -c 옵션으로 횟수 지정: ping -c 4 google.com
네트워크 연결 확인
curl <URL>
curl -X POST -H "Content-Type: application/json" -d '{"key":"value"}' http://localhost:3000/api. API 테스트 시 필수 도구입니다.
HTTP 요청 보내기
curl -I <URL>
응답 헤더(상태 코드, Content-Type 등)만 빠르게 확인합니다. 서버 상태 체크에 유용합니다.
HTTP 헤더만 조회
wget <URL>
wget -O filename.zip URL (파일명 지정 다운로드). curl과 달리 재시도, 재개 기능이 내장되어 있습니다.
파일 다운로드
nslookup <도메인>
도메인의 IP 주소를 확인합니다. DNS 문제 디버깅 시 사용합니다.
DNS 조회
dig <도메인>
nslookup보다 더 상세한 DNS 정보를 제공합니다. dig +short domain.com 으로 간단히 IP만 조회할 수 있습니다.
상세 DNS 조회
포트 & 연결
netstat -tlnp
-t(TCP), -l(LISTEN), -n(숫자 표시), -p(프로세스). 어떤 포트가 열려있고 어떤 프로세스가 사용 중인지 확인합니다.
열린 포트 확인
ss -tlnp
netstat의 최신 대체 명령어입니다. 더 빠르고 상세한 소켓 정보를 제공합니다.
열린 포트 확인 (최신)
lsof -i :<포트>
lsof -i :8080 → 8080 포트를 사용 중인 프로세스 PID와 이름을 확인합니다. 포트 충돌 해결의 첫 단계입니다.
포트 점유 프로세스 확인
ssh <유저>@<호스트>
ssh -i key.pem user@server.com (키 파일 사용), ssh -p 2222 user@host (포트 지정). 서버 관리의 기본입니다.
원격 서버 접속
scp <원본> <유저>@<호스트>:<경로>
scp file.txt user@server:/tmp/ (업로드), scp user@server:/tmp/file.txt . (다운로드). -r 옵션으로 디렉토리도 복사 가능합니다.
원격 파일 복사
시스템 정보
uname -a
커널 버전, 호스트명, 아키텍처 등 시스템 기본 정보를 한 줄로 보여줍니다.
시스템 정보 전체 출력
hostname
현재 서버의 호스트명을 출력합니다.
호스트명 확인
whoami
현재 로그인한 사용자명을 출력합니다. 권한 문제 디버깅의 첫 단계입니다.
현재 사용자 확인
uptime
서버가 얼마나 오래 켜져 있었는지, 현재 로드 평균(load average)을 보여줍니다.
서버 가동 시간
date
date +"%Y-%m-%d %H:%M:%S" (형식 지정 출력). 로그 타임스탬프 확인 시 서버 시간대를 체크하는 데 사용합니다.
현재 날짜/시간
cat /etc/os-release
Ubuntu, CentOS 등 배포판 이름과 버전을 확인합니다. 패키지 설치 시 배포판 확인이 필요할 때 사용합니다.
OS 버전 확인
메모리 & 디스크
free -h
-h 옵션으로 사람이 읽기 쉬운 단위(GB, MB)로 표시합니다. total, used, free, available을 한눈에 확인할 수 있습니다.
메모리 사용량 조회
df -h
마운트된 모든 파일시스템의 사용량/여유 공간을 표시합니다. 디스크 가득 참 문제 확인의 첫 단계입니다.
디스크 사용량 조회
du -sh <경로>
du -sh * (현재 디렉토리의 각 항목 크기). du -sh /var/log (로그 디렉토리 크기). 디스크 공간을 많이 차지하는 디렉토리를 찾을 때 사용합니다.
디렉토리 크기 확인
du -h --max-depth=1
어떤 디렉토리가 공간을 많이 차지하는지 빠르게 파악할 수 있습니다.
1단계 하위 디렉토리별 크기
압축 & 해제
tar -czf <이름>.tar.gz <경로>
-c(생성), -z(gzip), -f(파일명). 예: tar -czf backup.tar.gz ./src (src 디렉토리를 압축)
tar.gz 압축
tar -xzf <파일>.tar.gz
-x(추출), -z(gzip), -f(파일명). -C 옵션으로 특정 디렉토리에 해제: tar -xzf file.tar.gz -C /tmp/
tar.gz 해제
tar -xf <파일>
최근 tar는 압축 형식을 자동 감지합니다. tar.gz, tar.bz2, tar.xz 모두 이 명령으로 해제 가능합니다.
tar 아카이브 해제 (자동 감지)
zip -r <이름>.zip <경로>
zip -r archive.zip ./project (디렉토리를 zip으로 압축). Windows 사용자에게 파일을 전달할 때 주로 사용합니다.
zip 압축
unzip <파일>.zip
unzip file.zip -d /target/ (특정 디렉토리에 해제). unzip -l file.zip (내용 미리보기)
zip 해제
gzip <파일>
원본 파일이 .gz로 대체됩니다. gzip -k (원본 유지). gunzip으로 해제합니다.
gzip 단일 파일 압축
패키지 관리 (Ubuntu/Debian)
apt update
설치 가능한 패키지의 최신 버전 정보를 다운로드합니다. install 전에 항상 먼저 실행하세요.
패키지 목록 갱신
apt install <패키지>
sudo apt install nginx (nginx 설치). -y 옵션으로 확인 프롬프트를 건너뛸 수 있습니다.
패키지 설치
apt remove <패키지>
설정 파일은 남깁니다. apt purge로 설정 파일까지 완전히 제거할 수 있습니다.
패키지 제거
apt list --installed
apt list --installed | grep nginx (특정 패키지가 설치되어 있는지 확인)
설치된 패키지 목록
CentOS/RHEL은 apt 대신 yum 또는 dnf를 사용합니다. 예: dnf install nginx
모드 전환
i
현재 커서 위치에서 입력 모드로 전환합니다. 가장 기본적인 편집 시작 방법입니다.
커서 앞에서 입력 모드 시작
a
커서 다음 위치에서 입력 모드로 전환합니다. 줄 끝에 추가할 때는 A(Shift+a)를 사용하세요.
커서 뒤에서 입력 모드 시작
o
현재 줄 아래에 새 줄을 만들고 입력 모드로 전환합니다. O(Shift+o)는 위에 새 줄을 만듭니다.
아래에 새 줄 삽입 후 입력 모드
Esc
입력 모드에서 일반(Normal) 모드로 돌아갑니다. 가장 자주 누르는 키입니다.
일반 모드로 돌아가기
v
텍스트를 선택하는 비주얼 모드입니다. V(Shift+v)는 줄 단위 선택, Ctrl+v는 블록 선택입니다.
비주얼 모드 (텍스트 선택)
Vi는 모드 기반 편집기입니다. Normal(명령) → i로 Insert(입력) → Esc로 Normal → :로 Command(명령줄). 모드를 헷갈릴 때는 Esc를 여러 번 누르세요.
저장 & 종료
:w
현재 파일을 저장합니다. :w filename으로 다른 이름으로 저장할 수도 있습니다.
저장
:q
변경사항이 없을 때 종료합니다. 변경사항이 있으면 에러가 발생합니다.
종료
:wq
저장하고 종료합니다. 가장 많이 사용하는 조합입니다. ZZ(Shift+z 두 번)도 같은 동작입니다.
저장 후 종료
:q!
변경사항을 버리고 강제 종료합니다. 잘못 편집했을 때 원본 상태로 되돌리고 나갈 수 있습니다.
저장하지 않고 종료
:wq!
읽기 전용 파일도 강제로 저장하고 종료합니다. (권한이 있을 경우)
강제 저장 후 종료
커서 이동
hjkl
h(좌), j(하), k(상), l(우). 방향키도 동작하지만, 홈 포지션에서 벗어나지 않아 빠릅니다.
좌 / 하 / 상 / 우 이동
gg G
gg는 파일 맨 처음, G(Shift+g)는 파일 맨 끝으로 이동합니다. 10G로 10번째 줄로 이동할 수 있습니다.
파일 처음(gg) / 끝(G) 이동
0 $
0은 줄 맨 처음, $는 줄 맨 끝으로 이동합니다. ^는 첫 번째 비공백 문자로 이동합니다.
줄 처음(0) / 끝($) 이동
w b
w는 다음 단어, b는 이전 단어로 이동합니다. 한 글자씩 이동하는 것보다 훨씬 빠릅니다.
다음(w) / 이전(b) 단어 이동
:<줄번호>
:42 (42번째 줄로 이동). 에러 로그에서 줄 번호를 보고 바로 이동할 때 유용합니다.
특정 줄로 이동
편집
dd
현재 줄을 잘라냅니다 (삭제 + 클립보드 복사). 5dd로 5줄을 한 번에 삭제할 수 있습니다.
현재 줄 삭제 (잘라내기)
yy
현재 줄을 복사합니다. 3yy로 3줄을 복사할 수 있습니다. 비주얼 모드에서 선택 후 y도 가능합니다.
현재 줄 복사
p
커서 아래/뒤에 붙여넣습니다. P(Shift+p)는 커서 위/앞에 붙여넣습니다.
붙여넣기
u
마지막 작업을 취소합니다. 여러 번 누르면 계속 이전 상태로 돌아갑니다.
실행 취소 (Undo)
Ctrl+R
Undo를 취소합니다. u로 너무 많이 되돌렸을 때 사용합니다.
다시 실행 (Redo)
x
커서 위치의 한 글자를 삭제합니다. Delete 키와 같은 동작입니다.
한 글자 삭제
dw
커서부터 다음 단어 시작까지 삭제합니다. d$(줄 끝까지 삭제), d0(줄 처음까지 삭제)도 유용합니다.
단어 삭제
cw
단어를 삭제하고 바로 입력 모드로 전환합니다. 변수명이나 단어를 빠르게 바꿀 때 유용합니다.
단어 변경 (삭제 + 입력 모드)
.
마지막 편집 명령을 반복합니다. 같은 작업을 여러 곳에 적용할 때 매우 강력합니다.
마지막 명령 반복
검색 & 치환
/<패턴>
앞으로 검색합니다. n으로 다음 결과, N으로 이전 결과로 이동합니다. ?패턴은 뒤로 검색합니다.
앞으로 검색
n N
n은 다음 검색 결과, N(Shift+n)은 이전 검색 결과로 이동합니다.
다음(n) / 이전(N) 검색 결과
:%s/찾기/바꾸기/g
파일 전체에서 찾아 바꿉니다. /gc로 하면 각 항목마다 확인합니다. IDE의 Find & Replace와 같은 기능입니다.
전체 찾아 바꾸기
:set number
줄 번호를 표시합니다. :set nonumber로 해제. :set nu / :set nonu 약어도 사용 가능합니다.
줄 번호 표시
파이프 & 리다이렉션
|
ps aux | grep node | awk '{print $2}' → node 프로세스의 PID만 추출. 명령어를 체이닝하여 강력한 조합을 만들 수 있습니다.
명령어 출력을 다음 명령어 입력으로 전달
>
echo "hello" > file.txt (파일에 저장). 기존 내용을 완전히 덮어씁니다.
출력을 파일로 저장 (덮어쓰기)
>>
echo "new line" >> file.txt (기존 내용 뒤에 추가). 로그 기록 등에 사용합니다.
출력을 파일에 추가 (append)
2>&1
command > output.log 2>&1 (표준 출력과 에러를 모두 파일에 저장). 스크립트 로깅에 필수적인 패턴입니다.
에러 출력을 표준 출력에 합치기
< <파일>
mysql -u root < dump.sql (SQL 파일을 MySQL에 입력)
파일 내용을 입력으로 전달
자주 쓰는 조합
history | grep <키워드>
이전에 실행한 명령어를 키워드로 검색합니다. !123으로 123번 명령어를 재실행할 수 있습니다.
이전 명령어 검색
watch -n <초> <명령어>
watch -n 2 "df -h" (2초마다 디스크 사용량 확인). 모니터링에 유용합니다.
명령어 반복 실행
alias <이름>='<명령어>'
alias ll='ls -alh' (ll로 상세 목록 조회). ~/.bashrc에 추가하면 영구 적용됩니다.
명령어 별칭 만들기
Ctrl+R
터미널에서 Ctrl+R을 누르고 타이핑하면 이전 명령어를 실시간으로 검색합니다. 매우 자주 쓰는 단축키입니다.
히스토리 역검색
!!
sudo !! → 직전 명령어를 sudo로 다시 실행. 권한 부족으로 실패했을 때 유용합니다.
직전 명령어 재실행
command1 && command2
npm install && npm start (설치 성공 후 실행). 실패하면 다음 명령어를 실행하지 않습니다.
앞 명령 성공 시 다음 실행
환경 변수
env
현재 설정된 모든 환경 변수를 표시합니다. env | grep PATH 처럼 필터링해서 사용합니다.
환경 변수 전체 출력
echo $<변수명>
echo $PATH, echo $HOME, echo $USER 등. 변수 확인의 가장 빠른 방법입니다.
환경 변수 값 확인
export <변수>=<값>
export NODE_ENV=production. 현재 셸에서만 유효합니다. 영구 적용은 ~/.bashrc에 추가하세요.
환경 변수 설정
source ~/.bashrc
.bashrc나 .zshrc를 수정한 후 즉시 적용할 때 사용합니다. . ~/.bashrc 로도 실행 가능합니다.
